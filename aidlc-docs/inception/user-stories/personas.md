# User Personas

## Persona 1: 매장 방문 고객 (Customer)

| 항목 | 내용 |
|---|---|
| **이름** | 고객 (Customer) |
| **역할** | 매장에서 테이블 태블릿을 통해 음식을 주문하는 방문 고객 |
| **목표** | 빠르고 쉽게 메뉴를 탐색하고 주문하기 |
| **동기** | 직원 호출 없이 즉시 주문, 주문 상태 실시간 확인 |
| **기술 수준** | 다양함 (스마트폰/태블릿 기본 조작 가능) |
| **언어** | 한국어 또는 영어 |
| **특성** | 터치 기반 인터랙션, 짧은 체류 시간 내 빠른 주문 기대, 추가 주문 빈번 |
| **Pain Points** | 복잡한 UI, 느린 응답, 주문 상태 불확실성 |

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|---|---|
| **이름** | 관리자 (Admin) |
| **역할** | 매장 운영을 관리하는 사장/매니저 |
| **목표** | 실시간으로 주문을 모니터링하고 매장을 효율적으로 운영 |
| **동기** | 인건비 절감, 주문 누락 방지, 매장 운영 효율화 |
| **기술 수준** | 중간 (웹 브라우저 기본 조작 가능) |
| **언어** | 한국어 |
| **특성** | 장시간 대시보드 모니터링, 빠른 상태 변경 필요, 메뉴/테이블 관리 수행 |
| **Pain Points** | 주문 누락, 테이블 상태 파악 어려움, 메뉴 변경 번거로움 |
